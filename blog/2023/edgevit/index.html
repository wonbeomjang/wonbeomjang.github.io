EdgeViTs: Competing Light-weight CNNs on Mobile Devices with Vision Transformers

Introduction

ViTлҠ” globalн•ң representationмқ„ н•ҷмҠөн•ҳл©ҙм„ң imagenet benchmarkм—җм„ң м••лҸ„м Ғмқё м„ұлҠҘмқ„ лӮҙкі мһҲлӢӨ. н•ҳм§Җл§Ң self-attentionмқҙлқјлҠ” м—°мӮ°мқҳ л№„мҡ©мқҙ м»Өм„ң inference speedлӮҳ power efficiencyк°Җ л–Ём–ҙм§„лӢӨ. мқҙлҘј н•ҙкІ°н•ҳкё° мң„н•ң кё°мЎҙмқҳ м—°кө¬лҠ” нҒ¬кІҢ лӢӨмқҢкіј к°ҷм•ҳлӢӨ.

  1. Spatial Resolutionм—җм„ң hierarchical architectureмқ„ л§Ңл“Өм–ҙ м—°мӮ°лҹүмқ„ мӨ„мқёлӢӨ.
  2. Locally-grouped self-attention mechanismмқ„ мӮ¬мҡ©н•ңлӢӨ.
  3. Key, valueлҘј poolingн•ҳм—¬ subsamplingн•ңлӢӨ.

н•ҳм§Җл§Ң мқҙлҹ¬н•ң л°©лІ•мқҖ time costк°Җ мӨ‘мҡ”н•ң mobileмқҙлӮҳ edge platformм—җм„ң м¶©л¶„н•ҳм§Җ лӘ»н•ҳлӢӨ. л”°лқјм„ң м ҖмһҗлҠ” лӢӨмқҢмқҳ мҡ”мҶҢлҘј кі л Өн•ҳм—¬ EdgeViTлҘј м„Өкі„н–ҲлӢӨ.

Inference efficiency EdgeViTлҠ” on-deviceм—җм„ң мӮ¬мҡ©н•  м •лҸ„лЎң к°ҖліҚкі  м—җл„Ҳм§ҖлҘј м ҒкІҢ мӮ¬мҡ©н•ҙм•јн•ңлӢӨ. кё°мЎҙм—җлҠ” мқҙлҘј мёЎм •н•ҳкё°мң„н•ҙ FLOPSлҘј мЈјлЎң мӮ¬мҡ©н–Ҳм§Җл§Ң мқҙлҠ” latencyкіј energy consumptionмқ„ м ңлҢҖлЎң л°ҳмҳҒн•ҳм§Җ лӘ»н•ңлӢӨ. л”°лқјм„ң FLOPSлҠ” м¶”м •м№ҳлЎңл§Ң м°ёкі н•ҳкі  мӢӨм ң mobile deviceм—җм„ң лІӨм№ҳл§ҲнҒ¬лҘј м§„н–үн•ңлӢӨ.

2. Model size нҳ„лҢҖ мҠӨл§ҲнҠёнҸ°мқҖ RAMмқҙ 32GBмқјм •лҸ„лЎң л©”лӘЁлҰ¬лҹүмқҙ м¶©л¶„н•ҳлӢӨ. л”°лқјм„ң м ҲлҢҖм Ғмқё лӘЁлҚёнҒ¬кё°лҘј кі л Өн•ҳлҠ” кІғмқҖ нҳ„лҢҖ mobile deviceм—җлҠ” м Ғн•©н•ҳм§Җ м•Ҡм•„м„ң мқҙлҠ” нҒ¬кІҢ кі л Өн•ҳм§Җ м•ҠлҠ”лӢӨ.

3. Implementation Friendliness нҳ„мӢӨм ҒмңјлЎң implementationмқ„ н•ҳкё°мң„н•ҙм„ңлҠ” ONNX, TensorRT, TorchScriptл“ұ кё°мЎҙмқҳ frameworkмҷҖмқҳ нҳёнҷҳм„ұмқ„ кі л Өн•ҙм•јн•ңлӢӨ.

мқҙлҘј кі л Өн•ҳм—¬ м ҖмһҗлҠ” local-global-local bottleneckмқ„ м ңм•Ҳн–Ҳкі  мқҙлҠ” energy efficientн•ҳкі  inference speedлҸ„ л№ лҘҙлӢӨ.

Local-Global-Local-BottleNeck

ViTмқҳ м„ұлҠҘмқҙ мўӢмқҖ мқҙмң лҠ” globalн•ң representationмқ„ н•ҷмҠөн•ҳкё° мң лҰ¬н•ҳкё° л•Ңл¬ёмқҙлӢӨ. Multi-head self-attentionмқҙ мқҙлҘј к°ҖлҠҘн•ҳкІҢ л§Ңл“Өм—ҲмңјлӢҲ мқҙлҘј лӘЁмӮ¬н•ҳлҠ” м—°мӮ°мқ„ л§Ңл“ңлҠ” кІғмқ„ н•„мҲҳм ҒмқҙлӢӨ. л”°лқјм„ң м ҖмһҗлҠ” factorizationмқ„ нҶөн•ҙ н•ҙлӢ№ м—°мӮ°мқ„ 3к°Җм§ҖлЎң мӘјк°ң лӘЁмӮ¬н–ҲлӢӨ.

Local Aggregation

м—°мӮ°мқ„ к°ҖліҚкІҢ л§Ңл“Өкё° мң„н•ҙм„ңлҠ” м ҲлҢҖм Ғмқё м—°мӮ°лҹүмқ„ мӨ„м—¬м•ј н•ңлӢӨ. лӢӨн–үнһҲлҸ„ мқҙлҜём§Җм—җм„ңлҠ” мЈјліҖ н”Ҫм…ҖмқҖ л№„мҠ·н•ҳлӢӨлҠ” inductive biasк°Җ мЎҙмһ¬н•ңлӢӨ. л”°лқјм„ң м „мІҙ tokenм—җ лҢҖн•ҳм—¬ attentionмқ„ кі„мӮ°н•ҳм§Җ м•Ҡкі  мЈјліҖ н”Ҫм…Җмқҳ м •ліҙлҘј aggregationн•ЁмңјлЎңмҚЁ scopeлҘј мӨ„мқј мҲҳ мһҲлӢӨ. л”°лқјм„ң attention кі„мӮ°м—җ м•һм„ңм„ң locally proximate tokensмқҳ signalsмқ„ integrateн•ңлӢӨ. мқҙлҠ” кё°мЎҙ depth-wise separable convolutionмқ„ мӮ¬мҡ©н•ңлӢӨ.

Global Sparse Attention

мқҙм ң attentionм—җ лҢҖн•ң scopeмқ„ мӨ„мҳҖмңјлӢҲ long-range relationshipмқ„ кі„мӮ°н•ҙм•ј н•ңлӢӨ. н•ҳм§Җл§Ң лӘЁл“  нҶ нҒ°м—җ лҢҖн•ҙ self-attentionмқ„ мҲҳн–үн•ҳл©ҙ л№„мҡ©мқҙ нҒ¬лӢӨ. м ҖмһҗлҠ” к°Ғ \(r \times r\) local windowм—җм„ң н•ҳлӮҳмқҳ delegate tokenмқ„ м„ нғқн•ҳм—¬, delegate tokenл“Ө к°„м—җл§Ң self-attentionмқ„ мҲҳн–үн•ңлӢӨ. мқҙл•Ң \(r\)мқҖ sub-sampling rateмқҙлӢӨ.

кё°мЎҙ PVTsлӮҳ Twins-SVTsлҠ” keyмҷҖ valueл§Ң subsamplingн•ҳкі  queryлҠ” лӘЁл“  мң„м№ҳм—җм„ң кі„мӮ°н•ңлӢӨ. л°ҳл©ҙ EdgeViTлҠ” query, key, value лӘЁл‘җ delegate tokenм—җм„ңл§Ң кі„мӮ°н•ҳлҜҖлЎң м—°мӮ°лҹүмқҙ лҚ” м ҒлӢӨ. Sub-sampling rateлҠ” stageлі„лЎң лӢӨлҘҙкІҢ м„Өм •н•ңлӢӨ: \((r_1, r_2, r_3, r_4) = (4, 2, 2, 1)\). л§Ҳм§Җл§ү stageм—җм„ңлҠ” \(r = 1\)мқҙлҜҖлЎң standard MHSAмҷҖ лҸҷмқјн•ҳлӢӨ.

Local Propagation

мқҙм ң global contextual informationмқ„ кі„мӮ°н–ҲмңјлӢҲ local windowм—җ м „нҢҢн•ҙм•јн•ңлӢӨ. мқҙлҠ” к°„лӢЁн•ҳкІҢ transpose convolutionмңјлЎң кө¬м„ұн–ҲлӢӨ.

мң„мқҳ 3к°Җм§Җ м—°мӮ°мқ„ нҶөн•ҙ local-global-local bottleneckмқ„ кө¬м„ұн•ңлӢӨ. к·ёлҰ¬кі  мқҙлҠ” лӢӨмқҢкіј к°ҷмқҖ мӢқмңјлЎң м—°кІ°лҗңлӢӨ. \(X=LocalAgg(Norm(X_{in}))+X_{in}\) \(Y=FFN(Norm(X))+X\) \(Z=LocalProp(GlobalSparseAttn(Norm(Y)))+Y\) \(X_{out}=FFN(Norm(Z))+Z\) мқҙ л•Ң FFNмқҖ fully connected layer 2к°ңлЎң кө¬м„ұлҗҳм–ҙмһҲмңјл©° NormalizationмқҖ Layer Normalizationмқ„ мӮ¬мҡ©н•ңлӢӨ.

ImageNet

кІҪлҹү ViT лӘЁлҚёл“Ө мӨ‘м—җм„ң EdgeViTлҠ” accuracy-latency trade-offм—җм„ң мўӢмқҖ м„ұлҠҘмқ„ ліҙмқёлӢӨ. нҠ№нһҲ DeiT-Ti, PVT-Tiny л“ұ кё°мЎҙ кІҪлҹү ViT лҢҖл№„ л№„мҠ·н•ң м •нҷ•лҸ„м—җм„ң latencyк°Җ лӮ®лӢӨ. лӢӨл§Ң MobileNetV2, EfficientNet-B0 к°ҷмқҖ CNN лӘЁлҚёкіј л№„көҗн•ҳл©ҙ latencyм—җм„ңлҠ” м•„м§Ғ м—ҙм„ёмқёлҚ°, мқҙлҠ” ViTмқҳ attention м—°мӮ°мқҙ лӘЁл°”мқј н•ҳл“ңмӣЁм–ҙм—җм„ң м•„м§Ғ м¶©л¶„нһҲ мөңм Ғнҷ”лҗҳм§Җ м•Ҡм•ҳкё° л•Ңл¬ёмқҙлӢӨ.

Energy efficiency мёЎл©ҙм—җм„ң EdgeViTлҠ” нҠ№нһҲ л‘җл“ңлҹ¬м§„лӢӨ. м •нҷ•лҸ„ лҢҖл№„ мҶҢл№„ м „л Ҙмқ„ мёЎм •н•ҳл©ҙ EdgeViT-Sк°Җ к°ҖмһҘ нҡЁмңЁм Ғмқё лӘЁлҚёлЎң лӮҳнғҖлӮңлӢӨ. мқҙлҠ” Global Sparse Attentionмқҙ м „мІҙ нҶ нҒ°мқҙ м•„лӢҢ лҢҖн‘ң нҶ нҒ°(delegate token)л§Ң мӮ¬мҡ©н•ҳм—¬ м—°мӮ°лҹүлҝҗ м•„лӢҲлқј л©”лӘЁлҰ¬ м ‘к·јлҸ„ мӨ„мқҙкё° л•Ңл¬ёмқҙлӢӨ. л°°н„°лҰ¬ кё°л°ҳ л””л°”мқҙмҠӨм—җм„ңлҠ” latencyліҙлӢӨ м—җл„Ҳм§Җ нҡЁмңЁмқҙ лҚ” мӨ‘мҡ”н•ң кІҪмҡ°к°Җ л§ҺмңјлҜҖлЎң, EdgeViTмқҳ мӢӨмҡ©м Ғ к°Җм№ҳк°Җ мһҲлӢӨ.

мқҙ л…јл¬ёмқҳ мӢңмӮ¬м җ

EdgeViTк°Җ ліҙм—¬мЈјлҠ” н•өмӢ¬ л©”мӢңм§ҖлҠ” FLOPSл§ҢмңјлЎң лӘЁл°”мқј нҡЁмңЁм„ұмқ„ нҢҗлӢЁн•ҳл©ҙ м•Ҳ лҗңлӢӨлҠ” кІғмқҙлӢӨ. FLOPS, parameter мҲҳ, latency, energy consumptionмқҖ к°Ғк°Ғ лӢӨлҘё м§Җн‘ңмқҙл©°, мӢӨм ң л””л°”мқҙмҠӨм—җм„ң мёЎм •н•ң latencyмҷҖ м „л Ҙ мҶҢл№„к°Җ к°ҖмһҘ мӢ лў°н•  мҲҳ мһҲлҠ” кё°мӨҖмқҙлӢӨ. Local-Global-Local bottleneckмқҖ мқҙлЎ м ҒмңјлЎң мҡ°м•„н•ҳм§Җл§Ң, мӢӨм ңлЎңлҠ” CNN лҢҖл№„ лӘЁл°”мқј м¶”лЎ  мҶҚлҸ„м—җм„ң м•„м§Ғ gapмқҙ мһҲлӢӨ. мқҙлҠ” н•ҳл“ңмӣЁм–ҙ мөңм Ғнҷ”(NPU м§Җмӣҗ л“ұ)к°Җ м§„н–үлҗҳл©ҙ мўҒнҳҖм§Ҳ мҲҳ мһҲлҠ” л¶Җл¶„мқҙлӢӨ.
