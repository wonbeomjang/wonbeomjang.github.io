мқҙ кёҖмқҖ K8s мӢңлҰ¬мҰҲмқҳ лӢӨм„Ҝ лІҲм§ё кёҖмқҙлӢӨ. Kubernetes кё°мҙҲ к°ңл…җ(Pod, Deployment, Service л“ұ)мқҖ мқҙм „ кёҖмқ„ м°ёкі н•ҳмһҗ.

м§Ғм ‘ KubernetesлҘј кө¬м¶•н•ҳл©ҙ Control Plane м„Өм№ҳ, etcd л°ұм—…, мқёмҰқм„ң к°ұмӢ , лІ„м „ м—…к·ёл Ҳмқҙл“ң л“ұ нҒҙлҹ¬мҠӨн„° кҙҖлҰ¬м—җл§Ң мғҒлӢ№н•ң мҡҙмҳҒ л¶ҖлӢҙмқҙ мғқкёҙлӢӨ. AWS EKS(Elastic Kubernetes Service)лҠ” Control Plane кҙҖлҰ¬лҘј AWSм—җ мң„мһ„н•ҳкі , Rancher к°ҷмқҖ GUI кҙҖлҰ¬ лҸ„кө¬лҘј н•Ёк»ҳ мӮ¬мҡ©н•ҳл©ҙ kubectl лӘ…л №м–ҙлҘј лӘ°лқјлҸ„ Pod л°°нҸ¬, лЎңк·ё нҷ•мқё, мҠӨмјҖмқјл§Ғ л“ұ лҢҖл¶Җл¶„мқҳ мһ‘м—…мқ„ мӣ№ лёҢлқјмҡ°м Җм—җм„ң мҲҳн–үн•  мҲҳ мһҲлӢӨ. мҰү EKSк°Җ мқён”„лқј мҡҙмҳҒмқ„, Rancherк°Җ мқјмғҒ мҡҙмҳҒмқ„ лҚңм–ҙмЈјлҠ” мЎ°н•©мқҙлӢӨ.

мқҙ кёҖм—җм„ңлҠ” EKSмқҳ м•„нӮӨн…ҚмІҳл¶Җн„° л„ӨнҠёмӣҢнӮ№, ліҙм•Ҳ, мөңмӢ  кё°лҠҘ(Auto Mode, Pod Identity), л№„мҡ©, м—…к·ёл Ҳмқҙл“ң м „лһөк№Ңм§Җ мӢӨл¬ҙм—җм„ң н•„мҡ”н•ң н•өмӢ¬мқ„ м •лҰ¬н•ңлӢӨ.

1. EKS vs м§Ғм ‘ кө¬м¶•

н•ӯлӘ© м§Ғм ‘ кө¬м¶• EKS
Control Plane м§Ғм ‘ м„Өм№ҳВ·мҡҙмҳҒВ·м—…к·ёл Ҳмқҙл“ң AWSк°Җ кҙҖлҰ¬ (3 AZ кі к°Җмҡ©м„ұ)
etcd м§Ғм ‘ л°ұм—…В·ліөкө¬ AWSк°Җ кҙҖлҰ¬ (3 мқёмҠӨн„ҙмҠӨ л¶„мӮ°)
API Server м§Ғм ‘ HA кө¬м„ұ мөңмҶҢ 2к°ң мқёмҠӨн„ҙмҠӨ мһҗлҸҷ мҡҙмҳҒ
м—…к·ёл Ҳмқҙл“ң мҲҳлҸҷ (мң„н—ҳлҸ„ лҶ’мқҢ) мҪҳмҶ”/API н•ң лІҲм—җ мӢӨн–ү
AWS нҶөн•© мҲҳлҸҷ м—°лҸҷ IAM, VPC, ELB, EFS л“ұ л„ӨмқҙнӢ°лёҢ нҶөн•©
Worker Node м§Ғм ‘ кҙҖлҰ¬ м§Ғм ‘ кҙҖлҰ¬, Managed Node Group, лҳҗлҠ” мҷ„м „ мһҗлҸҷ(Auto Mode)
л№„мҡ© EC2 + мҡҙмҳҒ мқёкұҙл№„ $0.10/hr (~$73/мӣ”) + EC2

н•өмӢ¬мқҖ Control Plane кҙҖлҰ¬лҘј AWSм—җ л§ЎкёҙлӢӨлҠ” кІғмқҙлӢӨ. API Server, etcd, SchedulerлҘј м§Ғм ‘ мҡҙмҳҒн•  л•Ң л°ңмғқн•ҳлҠ” мһҘм•  лҢҖмқ‘, мқёмҰқм„ң кҙҖлҰ¬, лІ„м „ нҳёнҷҳм„ұ л¬ём ңлҘј мӢ кІҪ м“°м§Җ м•Ҡм•„лҸ„ лҗңлӢӨ.

2. EKS м•„нӮӨн…ҚмІҳ

EKS нҒҙлҹ¬мҠӨн„°лҠ” AWS кҙҖлҰ¬ мҳҒм—ӯ(Control Plane)кіј мӮ¬мҡ©мһҗ кҙҖлҰ¬ мҳҒм—ӯ(Data Plane)мңјлЎң лӮҳлүңлӢӨ.

2.1 Control Plane мғҒм„ё

кө¬м„ұ мҡ”мҶҢ м—ӯн•  л°°м№ҳ
API Server лӘЁл“  мҡ”мІӯмқҳ м§„мһ…м җ. kubectl, Rancher л“ұмқҙ м—¬кё°м—җ мҡ”мІӯ мөңмҶҢ 2к°ң, 3 AZ л¶„мӮ°
etcd нҒҙлҹ¬мҠӨн„° мғҒнғңлҘј м ҖмһҘн•ҳлҠ” л¶„мӮ° Key-Value DB 3к°ң мқёмҠӨн„ҙмҠӨ, 3 AZ л¶„мӮ°
Scheduler мғҲ Podмқ„ м–ҙлҠҗ л…ёл“ңм—җ л°°м№ҳн• м§Җ кІ°м • Control Plane лӮҙл¶Җ
Controller Manager Deployment, ReplicaSet л“ұ м»ЁнҠёлЎӨлҹ¬ лЈЁн”„ мӢӨн–ү Control Plane лӮҙл¶Җ

Control PlaneмқҖ нҒҙлҹ¬мҠӨн„°лі„лЎң мҷ„м „нһҲ кІ©лҰ¬лҗҳм–ҙ мһҲлӢӨ. лӢӨлҘё нҒҙлҹ¬мҠӨн„°лӮҳ AWS кі„м •кіј кіөмң н•ҳм§Җ м•ҠлҠ”лӢӨ. AWS кҙҖлҰ¬ VPCм—җ мң„м№ҳн•ҳл©°, ENI(Elastic Network Interface)лҘј нҶөн•ҙ мӮ¬мҡ©мһҗ VPCмҷҖ м—°кІ°лҗңлӢӨ.

2.2 API Server м—”л“ңнҸ¬мқёнҠё

лӘЁл“ң м ‘к·ј лІ”мң„ к¶ҢмһҘ мғҒнҷ©
Public мқён„°л„·м—җм„ң м ‘к·ј к°ҖлҠҘ к°ңл°ң/н…ҢмҠӨнҠё
Public + CIDR м ңн•ң н—Ҳмҡ©лҗң IPл§Ң м ‘к·ј мқјл°ҳ мҡҙмҳҒ
Private VPC лӮҙл¶Җм—җм„ңл§Ң м ‘к·ј ліҙм•Ҳ мӨ‘мӢң нҷҳкІҪ (к¶ҢмһҘ)

н”„лЎңлҚ•м…ҳм—җм„ңлҠ” Private м—”л“ңнҸ¬мқёнҠё лҳҗлҠ” Public + CIDR м ңн•ңмқ„ мӮ¬мҡ©н•ҳлҠ” кІғмқҙ к¶ҢмһҘлҗңлӢӨ.

3. Worker Node мҳөм…ҳ

EKSлҠ” 6к°Җм§Җ Worker Node мҳөм…ҳмқ„ м ңкіөн•ңлӢӨ. кҙҖлҰ¬ мҲҳмӨҖм—җ л”°лқј м„ нғқн•ҳл©ҙ лҗңлӢӨ.

мҳөм…ҳ кҙҖлҰ¬ мҲҳмӨҖ м Ғн•©н•ң кІҪмҡ°
EKS Auto Mode AWS мҷ„м „ кҙҖлҰ¬ мҡҙмҳҒ мҳӨлІ„н—Өл“ң мөңмҶҢнҷ”
Fargate м„ңлІ„лҰ¬мҠӨ мқён”„лқј кҙҖлҰ¬ л¶Ҳн•„мҡ” нҷҳкІҪ
Karpenter (мһҗмІҙ м„Өм№ҳ) л°ҳмһҗлҸҷ м„ёл°Җн•ң мҠӨмјҖмқјл§Ғ м ңм–ҙ
Managed Node Group AWS л¶Җл¶„ кҙҖлҰ¬ мһҗлҸҷнҷ”мҷҖ м ңм–ҙмқҳ к· нҳ•
Self-Managed Node мӮ¬мҡ©мһҗ мҷ„м „ кҙҖлҰ¬ мҷ„м „н•ң м»ӨмҠӨн„°л§Ҳмқҙм§• н•„мҡ”
Hybrid Nodes н•ҳмқҙлёҢлҰ¬л“ң мҳЁн”„л ҲлҜёмҠӨ + нҒҙлқјмҡ°л“ң

3.1 Managed Node Group

к°ҖмһҘ ліҙнҺём Ғмқё м„ нғқмқҙлӢӨ. AWSк°Җ EC2 мқёмҠӨн„ҙмҠӨмқҳ н”„лЎңл№„м ҖлӢқ, нҢЁм№ӯ, м—…лҚ°мқҙнҠёлҘј кҙҖлҰ¬н•ңлӢӨ.

  • Auto Scaling Group кё°л°ҳмңјлЎң л…ёл“ң мҲҳлҘј мһҗлҸҷ мЎ°м Ҳ
  • Launch TemplateмңјлЎң мқёмҠӨн„ҙмҠӨ нғҖмһ…, AMI, userdata л“ұ м»ӨмҠӨн„°л§Ҳмқҙм§• к°ҖлҠҘ
  • м—…лҚ°мқҙнҠё мӢң мһҗлҸҷ л“ңл Ҳмқё: л…ёл“ң м—…лҚ°мқҙнҠё мӢң кё°мЎҙ Podмқ„ лӢӨлҘё л…ёл“ңлЎң мқҙлҸҷ нӣ„ көҗмІҙ

3.2 Fargate

Pod лӢЁмң„мқҳ м„ңлІ„лҰ¬мҠӨ мӢӨн–үмқҙлӢӨ. л…ёл“ңлҘј кҙҖлҰ¬н•  н•„мҡ”к°Җ м „нҳҖ м—ҶлӢӨ.

нҠ№м§• м„ӨлӘ…
кіјкёҲ vCPU-мҙҲ + Memory-мҙҲ (мөңмҶҢ 1л¶„)
мҠӨнҶ лҰ¬м§Җ 20GB ephemeral (л¬ҙлЈҢ)
м ңн•ң DaemonSet мӢӨн–ү л¶Ҳк°Җ, GPU лҜём§Җмӣҗ
м Ғн•©н•ң мҡ©лҸ„ к°„н—җм Ғ л°°м№ҳ мһ‘м—…, к°ҖлІјмҡҙ л§ҲмқҙнҒ¬лЎңм„ңл№„мҠӨ

Fargate ProfileлЎң м–ҙл–Ө Podмқҙ Fargateм—җм„ң мӢӨн–үлҗ м§Җ м§Җм •н•ңлӢӨ (Namespace + Label мЎ°н•©).

3.3 EKS Auto Mode (2024.12 GA)

2024л…„ re:Inventм—җм„ң л°ңн‘ңлҗң к°ҖмһҘ нҒ° ліҖнҷ”лӢӨ. м»ҙн“ЁнҢ…, мҠӨнҶ лҰ¬м§Җ, л„ӨнҠёмӣҢнӮ№мқ„ нҸ¬н•Ён•ң нҒҙлҹ¬мҠӨн„° мқён”„лқј кҙҖлҰ¬лҘј мһҗлҸҷнҷ”н•ңлӢӨ.

лӮҙл¶Җм ҒмңјлЎң Karpenterк°Җ EKS Control Planeм—җ нҶөн•©лҗң кІғмқҙлӢӨ. мӮ¬мҡ©мһҗк°Җ KarpenterлҘј м§Ғм ‘ м„Өм№ҳВ·кҙҖлҰ¬н•  н•„мҡ” м—Ҷмқҙ лҸҷмқјн•ң кё°лҠҘмқ„ мӮ¬мҡ©н•  мҲҳ мһҲлӢӨ.

кё°лҠҘ м„ӨлӘ…
м»ҙн“ЁнҢ… мҳӨнҶ мҠӨмјҖмқјл§Ғ Spot + On-Demand мһҗлҸҷ нҳјн•©, мӣҢнҒ¬лЎңл“ңм—җ л§һлҠ” мқёмҠӨн„ҙмҠӨ нғҖмһ… мһҗлҸҷ м„ нғқ
лӮҙмһҘ м• л“ңмҳЁ VPC CNI, EBS CSI, CoreDNS, kube-proxy л“ұ мһҗлҸҷ кҙҖлҰ¬
ліҙм•Ҳ Bottlerocket AMI кё°л°ҳ, SELinux, мқҪкё°м „мҡ© лЈЁнҠё нҢҢмқјмӢңмҠӨн…ң
мһҗлҸҷ м—…лҚ°мқҙнҠё PodDisruptionBudgetмқ„ мЎҙмӨ‘н•ҳл©° л…ёл“ң мһҗлҸҷ м—…лҚ°мқҙнҠё
GPU м§Җмӣҗ GPU н”Ңлҹ¬к·ёмқё мһҗлҸҷ м„Өм№ҳ

2025л…„ м¶”к°Җлҗң кё°лҠҘ:

  • EC2 On-Demand Capacity Reservations (ODCR) л°Ҹ Capacity Blocks for ML м§Җмӣҗ
  • лі„лҸ„ Pod м„ңлёҢл„· м§ҖмӣҗмңјлЎң мқён”„лқј/м•ұ нҠёлһҳн”Ҫ л¶„лҰ¬
  • AWS KMS м•”нҳёнҷ” (ephemeral + root ліјлҘЁ)
  • Forward Proxy м§Җмӣҗ

Auto Mode vs Managed Node Group

н•ӯлӘ© Auto Mode Managed Node Group
мқёмҠӨн„ҙмҠӨ нғҖмһ… м„ нғқ мһҗлҸҷ (мӣҢнҒ¬лЎңл“ң кё°л°ҳ) мҲҳлҸҷ м§Җм •
мҠӨмјҖмқјл§Ғ мҰүмӢң (Karpenter кё°л°ҳ) Cluster Autoscaler (лҠҗлҰј)
AMI кҙҖлҰ¬ мһҗлҸҷ (Bottlerocket) Amazon Linux 2/Bottlerocket м„ нғқ
м• л“ңмҳЁ кҙҖлҰ¬ лӮҙмһҘ лі„лҸ„ м„Өм№ҳВ·кҙҖлҰ¬
м»ӨмҠӨн„°л§Ҳмқҙм§• м ңн•ңм Ғ (NodePool/NodeClass) Launch TemplateмңјлЎң мһҗмң лЎңмӣҖ
л№„мҡ© EC2 + кҙҖлҰ¬ мҲҳмҲҳлЈҢ EC2л§Ң

м„ нғқ кё°мӨҖ: мҡҙмҳҒ к°„мҶҢнҷ”лҘј мӣҗн•ҳл©ҙ Auto Mode, м„ёл°Җн•ң м ңм–ҙк°Җ н•„мҡ”н•ҳл©ҙ Managed Node Group.

3.4 Karpenter вҖ” м§ҖлҠҘнҳ• л…ёл“ң мҳӨнҶ мҠӨмјҖмқјл§Ғ

KarpenterлҠ” AWSк°Җ л§Ңл“  мҳӨн”ҲмҶҢмҠӨ л…ёл“ң мҳӨнҶ мҠӨмјҖмқјлҹ¬лЎң, Cluster AutoscalerлҘј лҢҖмІҙн•ңлӢӨ. Podмқҙ Pending мғҒнғңк°Җ лҗҳл©ҙ мӣҢнҒ¬лЎңл“ңм—җ л§һлҠ” мөңм Ғмқҳ мқёмҠӨн„ҙмҠӨлҘј м§Ғм ‘ м„ нғқн•ҳм—¬ мҲҳ мҙҲ лӮҙм—җ л…ёл“ңлҘј н”„лЎңл№„м ҖлӢқн•ңлӢӨ.

Cluster Autoscaler vs Karpenter

н•ӯлӘ© Cluster Autoscaler Karpenter
мҠӨмјҖмқјл§Ғ лӢЁмң„ Node Group (лҜёлҰ¬ м •мқҳлҗң мқёмҠӨн„ҙмҠӨ нғҖмһ…) к°ңлі„ мқёмҠӨн„ҙмҠӨ (мӣҢнҒ¬лЎңл“ңм—җ л§һм¶° мһҗлҸҷ м„ нғқ)
мқёмҠӨн„ҙмҠӨ м„ нғқ Node Groupм—җ м§Җм •лҗң нғҖмһ…л§Ң мӮ¬мҡ© мҲҳл°ұ к°ң мқёмҠӨн„ҙмҠӨ нғҖмһ… мӨ‘ мөңм Ғ м„ нғқ
мҠӨмјҖмқј м•„мӣғ мҶҚлҸ„ ASG кё°л°ҳ, мҲҳ л¶„ мҲҳ мҙҲ (EC2 Fleet API м§Ғм ‘ нҳём¶ң)
Spot мІҳлҰ¬ Node Groupлі„ Spot/On-Demand л¶„лҰ¬ н•ҳлӮҳмқҳ NodePoolм—җм„ң мһҗлҸҷ нҳјн•©
л…ёл“ң м •лҰ¬ мӮ¬мҡ©лҘ  лӮ®мқҖ Node Group м¶•мҶҢ Podмқ„ мһ¬л°°м№ҳн•ҳкі  л№Ҳ л…ёл“ң мҰүмӢң мў…лЈҢ (Consolidation)
GPU м§Җмӣҗ Node Group лі„лҸ„ кө¬м„ұ н•„мҡ” nvidia.com/gpu мҡ”мІӯн•ҳл©ҙ мһҗлҸҷмңјлЎң GPU мқёмҠӨн„ҙмҠӨ м„ нғқ

Karpenterк°Җ LLM мӣҢнҒ¬лЎңл“ңм—җ мўӢмқҖ мқҙмң 

  1. GPU мқёмҠӨн„ҙмҠӨ мһҗлҸҷ м„ нғқ: Podмқҙ nvidia.com/gpu: 4лҘј мҡ”мІӯн•ҳл©ҙ, Karpenterк°Җ p4d.24xlarge, g5.12xlarge л“ұ мӮ¬мҡ© к°ҖлҠҘн•ң GPU мқёмҠӨн„ҙмҠӨ мӨ‘ к°ҖмһҘ м Җл ҙн•ң кІғмқ„ мһҗлҸҷмңјлЎң м„ нғқн•ңлӢӨ
  2. Spot + On-Demand нҳјн•©: н•ҷмҠө Jobм—җлҠ” Spot, м„ңл№ҷ м„ңлІ„м—җлҠ” On-DemandлҘј н•ҳлӮҳмқҳ NodePoolм—җм„ң мһҗлҸҷ м Ғмҡ©
  3. Consolidation: лІӨм№ҳл§ҲнҒ¬ Jobмқҙ лҒқлӮҳл©ҙ л№Ҳ GPU л…ёл“ңлҘј мҲҳ мҙҲ лӮҙм—җ мў…лЈҢ вҶ’ л№„мҡ© м Ҳк°җ
  4. л№ лҘё мҠӨмјҖмқјл§Ғ: Pending Pod к°җм§Җ нӣ„ мҲҳ мҙҲ лӮҙ л…ёл“ң н”„лЎңл№„м ҖлӢқ вҶ’ лҢҖкё° мӢңк°„ мөңмҶҢнҷ”

Karpenter н•өмӢ¬ лҰ¬мҶҢмҠӨ
