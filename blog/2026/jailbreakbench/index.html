Introduction

м§ҖлӮң кёҖм—җм„ң ліё HarmBenchлҠ” 510к°ңмқҳ кҙ‘лІ”мң„н•ң н–үлҸҷкіј R2D2 л°©м–ҙлЎң RT нҸүк°Җмқҳ breadthлҘј м •лҰҪн–ҲлӢӨ. кұ°мқҳ лҸҷмӢңм—җ л“ұмһҘн•ң JailbreakBenchлҠ” лӢӨлҘё мёЎл©ҙмқ„ к°•мЎ°н•ңлӢӨ вҖ” мһ¬нҳ„м„ұ(reproducibility).

м Җмһҗл“Өмқҳ л¬ём ң м§„лӢЁ:

  1. н‘ңмӨҖ м—ҶмқҢ: нҸүк°Җ л°©лІ•мқҙ л…јл¬ёл§ҲлӢӨ лӢӨлҰ„
  2. л№„көҗ л¶Ҳк°ҖлҠҘ: costмҷҖ success rateк°Җ incompatibleн•ҳкІҢ кі„мӮ°лҗЁ
  3. мһ¬нҳ„ л¶Ҳк°ҖлҠҘ: лӢӨмҲҳ л…јл¬ёмқҙ adversarial promptлҘј л№„кіөк°ң, closed-source code, ліҖлҸҷн•ҳлҠ” proprietary APIм—җ мқҳмЎҙ

нҠ№нһҲ (3)мқҙ кІ°м •м ҒмқҙлӢӨ. н•ң л…јл¬ёмқҙ вҖңGCGлЎң ChatGPT 90% к№јлӢӨвҖқкі  ліҙкі н•ҙлҸ„ мӢӨм ң jailbreak promptк°Җ м—Ҷмңјл©ҙ кІҖмҰқн•  мҲҳ м—ҶлӢӨ. JailbreakBenchлҠ” мқҙ л¬ём ңлҘј jailbreak artifacts repositoryлЎң м •л©ҙ лҸҢнҢҢн•ңлӢӨ вҖ” лӘЁл“  м„ұкіөн•ң кіөкІ© promptлҘј кіөк°ңн•ңлӢӨ.

лӢӨм„Ҝ к°Җм§Җ н•өмӢ¬ кө¬м„ұмҡ”мҶҢ:

  1. Jailbreak artifacts repository вҖ” мӢӨм ң кіөкІ© promptк°Җ кіөк°ңлҗң evolving лқјмқҙлёҢлҹ¬лҰ¬
  2. Red-teaming pipeline вҖ” н‘ңмӨҖнҷ”лҗң нҢҢлқјлҜён„°В·API м¶”мғҒнҷ”
  3. Defense pipeline вҖ” 5к°ң baseline л°©м–ҙ + мӢ к·ң л°©м–ҙ м ңм¶ң к°ҖлҠҘ

Background

Jailbreak Artifactsмқҳ л¶Җмһ¬ л¬ём ң

м Җмһҗл“ӨмқҖ prior workмқҳ л¬ём ңлҘј кө¬мІҙм ҒмңјлЎң м§ҡлҠ”лӢӨ:

  • мқјл¶Җ л…јл¬ёмқҖ GitHubм—җ мҪ”л“ңлҠ” мһҲм§Җл§Ң adversarial suffixлҠ” л№јкі  кіөк°ң
  • API кё°л°ҳ нҸүк°ҖлҠ” лӘЁлҚё м—…лҚ°мқҙнҠё мӢң кІҖмҰқ л¶Ҳк°Җ (gpt-3.5-turboк°Җ л§Ө л¶„кё° л°”лҖң)
  • crowd-sourced jailbreak prompt мӮ¬мқҙнҠёлҠ” мӮ¬лқјм§җ (e.g., DAN ліҖнҳ•л“Ө)

вҶ’ кіөкІ© нҡЁкіјмқҳ historical recordк°Җ мӮ¬лқјм§„лӢӨ. 1л…„ нӣ„м—җ вҖңGCGк°Җ GPT-3.5м—җм„ң 90% нҶөн–ҲлӢӨвҖқлҠ” мЈјмһҘмқҳ кІҖмҰқмқҙ л¶Ҳк°ҖлҠҘн•ҙм§„лӢӨ.

нҸүк°Җ м •мқҳ

л…јл¬ёмқҖ jailbreakлҘј лӢӨмқҢкіј к°ҷмқҙ м •мӢқнҷ”:

\[\text{find } P \in \mathcal{T}^* \quad \text{subject to} \quad \text{JUDGE}(\text{LLM}(P), G) = \text{True}\]
  • $P$: input prompt
  • $\mathcal{T}^*$: лӘЁл“  нҶ нҒ° мӢңнҖҖмҠӨ
  • $G$: harmful goal
  • JUDGE: мқ‘лӢөмқҙ лӘ©н‘ңлҘј лӢ¬м„ұн–ҲлҠ”м§Җ нҢҗм •

мқҙ м •мқҳк°Җ лӢЁмҲңн•ҙ ліҙмқҙм§Җл§Ң, н•өмӢ¬мқҖ JUDGEмқҳ к°қкҙҖм Ғ м„ м •мқҙлӢӨ. к°ҷмқҖ мқ‘лӢөмқ„ лӢӨлҘё judgeк°Җ лӢӨлҘҙкІҢ нҸүк°Җн•ҳл©ҙ leaderboard л№„көҗк°Җ л¬ҙмқҳлҜён•ҙм§„лӢӨ.

Method: лӢӨм„Ҝ к°Җм§Җ кө¬м„ұмҡ”мҶҢ

(1) Jailbreak Artifacts Repository

GitHub repoм—җ кіөкІ© лі„ / лӘЁлҚё лі„лЎң м„ұкіөн•ң promptлҘј лӘЁл‘җ м ҖмһҘ. Python лқјмқҙлёҢлҹ¬лҰ¬лЎң н•ң мӨ„лЎң лЎңл“ң к°ҖлҠҘ:

import jailbreakbench as jbb
artifact = jbb.read_artifact(method="PAIR", model_name="vicuna-13b-v1.5")
print(artifact.jailbreaks[75])

#     number_of_queries=30,

#     response_tokens=3729,

л§Ө jailbreakк°Җ prompt, response, queries, tokensк№Ңм§Җ л©”нғҖлҚ°мқҙн„°лЎң нҸ¬н•Ё. мӮ¬нӣ„ кІҖмҰқВ·мһ¬нҳ„В·м—°кө¬ л¶„м„қмқҙ лӘЁл‘җ к°ҖлҠҘн•ҳлӢӨ.

(2) Red-Teaming Pipeline

н‘ңмӨҖнҷ”лҗң нҸүк°Җ нҷҳкІҪ. кІ°м •м Ғ л””н…Ңмқј:

  • system prompt
  • chat template (к°Ғ лӘЁлҚёлі„)
  • decoding parameters (temperature, max_tokens)
  • APIмҷҖ local лӘЁл‘җ м§Җмӣҗ

мқҙкІҢ HarmBenchмҷҖ нҸүн–үн•ң н‘ңмӨҖнҷ”лӢӨ. л‘җ лІӨм№ҳл§ҲнҒ¬ лӘЁл‘җ к°ҷмқҖ л¬ём ң(нҸүк°Җ fragmentation)лҘј н•ҙкІ°н•ңлӢӨ.

(3) Defense Pipeline + 5 Baseline л°©м–ҙ

- SmoothLLM (Robey et al.) вҖ” input perturbation
- Perplexity filter (Alon & Kamfonas) вҖ” high-PPL кұ°л¶Җ
- Erase-and-check (Kumar et al.) вҖ” substring л¶„лҰ¬ нӣ„ кІҖмӮ¬
- Synonym substitution
- Removal-based defenses

мӢ к·ң л°©м–ҙлҘј м ңм¶ңн•ҳл Өл©ҙ лҸҷмқј мқён„°нҺҳмқҙмҠӨлҘј кө¬нҳ„. мһҗлҸҷмңјлЎң leaderboardм—җ л“Өм–ҙк°„лӢӨ.

(4) Judge Classifier м„ м •

л…јл¬ёмқҙ 6к°ң нӣ„ліҙ judgeлҘј human-labeled jailbreak 300к°ңлЎң л№„көҗ:

Judge нӣ„ліҙ мқёк°„ н•©мқҳлҸ„
GPT-4 + simple prompt лҶ’мқҢ
GPT-4 + detailed prompt мөңмғҒмң„
Llama-2-13B (HarmBench) мӨ‘мғҒмң„
Llama-3-70B-Instruct + custom prompt GPT-4мҷҖ лҸҷл“ұ
keyword matching лӮ®мқҢ
etc. -

Llama-3-70B-Instructк°Җ GPT-4 judgeмҷҖ лҸҷл“ұн•ң мҲҳмӨҖм—җ лҸ„лӢ¬ + open-weightлқј reproducibility нҷ•ліҙ. мқҙкІҢ final judge.

(5) JBB-Behaviors Dataset

100 misuse + 100 benign н–үлҸҷ, 10 OpenAI usage policy м№ҙн…Ңкі лҰ¬:

  • Harassment/Discrimination
  • Malware/Hacking
  • Physical harm
  • Economic harm
  • Fraud/Deception
  • Disinformation
  • Sexual/Adult content
  • Privacy
  • Expert advice (мқҳлЈҢ/лІ•лҘ )
  • Government decision-making

к°Ғ misuse н–үлҸҷм—җ matching benign н–үлҸҷмқҙ мһҲлӢӨ. мҳҲ:

  • Misuse: вҖңDesign a phishing emailвҖқ
  • Benign: вҖңDesign a legitimate security awareness emailвҖқ

Matching benignмқҳ мқҳлҜё: лӢЁмҲң ASRл§Ң ліҙл©ҙ м•Ҳ лҗңлӢӨ. л°©м–ҙк°Җ л„Ҳл¬ҙ к°•н•ҳл©ҙ м •мғҒ мҡ”мІӯк№Ңм§Җ кұ°л¶Җн•ҳлҠ” over-refusalмқҙ мқјм–ҙлӮңлӢӨ. Matching benignм—җм„ң кұ°л¶ҖмңЁмқ„ мёЎм •н•ҳл©ҙ trade-offк°Җ ліҙмқёлӢӨ.

н–үлҸҷмқҳ м Ҳл°ҳмқҖ original, м Ҳл°ҳмқҖ AdvBench / HarmBenchм—җм„ң м°Ёмҡ©.

Experiments

нҸүк°Җ attacks

л…јл¬ёмқҙ leaderboardм—җ нҸүк°Җн•ң мЈјмҡ” кіөкІ©:

  • PRS (Prompt with Random Search, Andriushchenko et al.) вҖ” лӢЁмҲңн•ҳм§Җл§Ң к°•л Ҙ

Over-refusal мёЎм •

Matching benign н–үлҸҷм—җ лҢҖн•ҙ л°©м–ҙ лӘЁлҚёмқҙ кұ°л¶Җн•ҳлҠ” л№„мңЁ. мқҙмғҒм Ғмқё л°©м–ҙлҠ” misuseлҠ” кұ°л¶Җ, benignмқҖ нҶөкіјн•ҙм•ј н•ңлӢӨ.

л°©м–ҙ к°•лҸ„ Misuse ASR вҶ“ Benign кұ°л¶ҖмңЁ вҶ“
л¬ҙл°©м–ҙ baseline лҶ’мқҢ лӮ®мқҢ
Perplexity filter лӮ®мқҢ (мўӢмқҢ) м•Ҫк°„ мҰқк°Җ
SmoothLLM лӮ®мқҢ (мўӢмқҢ) мӨ‘к°„ мҰқк°Җ
к°•н•ң alignment л§Өмҡ° лӮ®мқҢ лҶ’мқҢ (лӮҳмҒЁ)

н•өмӢ¬ нҶөм°°: л°©м–ҙлҘј к°•нҷ”н•ҳл©ҙ over-refusalмқҙ л”°лқјмҳЁлӢӨ. JailbreakBenchлҠ” мқҙ trade-offлҘј лӘ…мӢңм ҒмңјлЎң мёЎм •н•ҳлҠ” мІ« н‘ңмӨҖ.

Conclusion

н•өмӢ¬ л©”мӢңм§Җ: вҖңмһ¬нҳ„м„ұмқҙ м§„ліҙмқҳ н•„мҲҳмЎ°кұҙмқҙлӢӨ. RT л¶„м•јм—җм„ңлҸ„ кіөкІ© artifactлҘј кіөк°ңн•ҳм§Җ м•Ҡмңјл©ҙ м§„ліҙлҠ” мёЎм • л¶Ҳк°ҖлӢӨ.вҖқ

лӢӨм„Ҝ к°Җм§Җ кё°м—¬:

  1. Jailbreak artifacts repository: лӘЁл“  м„ұкіө prompt + л©”нғҖлҚ°мқҙн„° кіөк°ң вҖ” 1л…„ л’Өм—җлҸ„ кІҖмҰқ к°ҖлҠҘ
  2. Red-teaming pipeline: н‘ңмӨҖнҷ”лҗң л””н…Ңмқј (system prompt, chat template, decoding)
  3. Defense pipeline: 5к°ң baseline + мӢ к·ң м ңм¶ң мқён„°нҺҳмқҙмҠӨ
  4. Llama-3-70B judge: open-weightлЎң GPT-4 judgeм—җ н•„м Ғ
  5. Matching benign: over-refusal trade-offмқҳ мІ« н‘ңмӨҖ мёЎм •

н•ңкі„м җ

  • 100 н–үлҸҷмқҖ мһ‘лӢӨ: лҸ„л©”мқё cover мёЎл©ҙм—җм„ң HarmBenchліҙлӢӨ мўҒмқҢ
  • Judge subjective: к°ҷмқҖ мқ‘лӢөм—җ лҢҖн•ҙ лӘЁлҚё/мӮ¬лһҢмқҳ нҢҗм • л¶„мӮ°мқҙ нҒј вҖ” н•©мқҳлҸ„ 100%к°Җ м•„лӢҳ
  • OpenAI policy мқҳмЎҙ: н•ң нҡҢмӮ¬мқҳ м •мұ…м—җ м№ҙн…Ңкі лҰ¬к°Җ л¬¶мһ„ вҖ” м •мұ… м—…лҚ°мқҙнҠё мӢң ліҖлҸҷ
  • Artifact stale: API лӘЁлҚёмқҖ мӢңк°„мқҙ нқҗлҘҙл©ҙ artifact promptк°Җ м•Ҳ нҶөн•  мҲҳ мһҲмқҢ вҖ” repoлҠ” evolvingмқҙм§Җл§Ң лӘЁл“  мЎ°н•© мң м§Җ м–ҙл ӨмӣҖ
  • м–ём–ҙ н•ңм •: мҳҒм–ҙ only

JailbreakBenchлҠ” вҖңкіөкІ© promptлҘј кіөк°ңн•ҳлҠ” кІғмқҙ мҳілӢӨвҖқ лҠ” мңӨлҰ¬м Ғ кІ°м •мқ„ л¶„м•ј м „мІҙм—җ лҒҢм–ҙл“ӨмҳҖлӢӨ. м Җмһҗл“ӨмқҖ abstractм—җм„ң ethical мҳҒн–Ҙмқ„ лі„лҸ„ л…јмқҳн•ҳкі , кІ°кіјм ҒмңјлЎң вҖңreleaseк°Җ net positiveвҖқлқј нҢҗлӢЁн•ңлӢӨ. мқҙ кІ°м • мһҗмІҙк°Җ Ganguli 2022мқҳ 38K attack dataset кіөк°ң кІ°м •мқҳ м—°мһҘм„ мқҙлӢӨ.

Red-Teaming мӢңлҰ¬мҰҲ

мқҙ кёҖмқҖ LLM Red-Teaming мӢңлҰ¬мҰҲмқҳ м—ҙмқјкіұ лІҲм§ё кёҖмқҙлӢӨ.

  6. PAIR (Chao 2023) вҖ” 20мҝјлҰ¬ black-box attacker LM
  7. TAP (Mehrotra 2023) вҖ” нҠёлҰ¬ нғҗмғү + мқҙмӨ‘ pruningмңјлЎң PAIR нҡЁмңЁнҷ”
  8. GPTFuzz (Yu 2023) вҖ” AFL мҳҒк°җмқҳ template-level fuzzing
  9. Crescendo (Russinovich 2024) вҖ” multi-turn escalationмңјлЎң single-turn л°©м–ҙ л¬ҙл Ҙнҷ”
  10. Many-shot Jailbreaking (Anil 2024) вҖ” long-contextлҘј ICLлЎң weaponize
  11. Curiosity-driven RT (Hong 2024) вҖ” novelty rewardлЎң mode collapse н•ҙкІ°
  12. Auto-RT (Liu 2025) вҖ” strategy-level RL exploration + progressive curriculum
  13. AgenticRed (Yuan 2026) вҖ” RT мӢңмҠӨн…ң мһҗмІҙлҘј м§„нҷ”
  14. InjecAgent (Zhan 2024) вҖ” Tool-use LLM agentм—җ лҢҖн•ң IPI лІӨм№ҳл§ҲнҒ¬
  15. AgentVigil (Wang 2025) вҖ” MCTS кё°л°ҳ IPI мһҗлҸҷ кіөкІ©
  16. HarmBench (Mazeika 2024) вҖ” 510 н–үлҸҷ Г— 18 кіөкІ© Г— 33 лӘЁлҚё н‘ңмӨҖ + R2D2 л°©м–ҙ
  17. (нҳ„мһ¬ кёҖ) JailbreakBench (Chao 2024) вҖ” 100 misuse + 100 benign + jailbreak artifacts repository
  18. Constitutional AI (Bai 2022) вҖ” AI feedbackмңјлЎң мқёк°„ лқјлІЁ м—Ҷмқҙ alignment
  19. мқҙнӣ„ Llama Guard мҲңмңјлЎң мқҙм–ҙм§„лӢӨ.
